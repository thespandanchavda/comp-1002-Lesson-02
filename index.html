Semantic HTML This website is intended to help inform interested parties about the importance of semantic HTML. Why is it important? Semantic HTML provides many benefits to people, devices, and software reading our HTML documents, and how they're able to comprehend them. How to apply Semantic HTML to a document 1. Start by pasting just the text of the content into the body of a new HTML document 2. Determine the major components of your content o Main Header - What is your heading content? o Main - What is the main content? 3. Once you determined the major components, determine if there are any sub-components 4. Now that you have your sub components, review your content and see if any further semantic tweaking could help refine your content and provide more meaning How does it help accessibility? Structural, semantic HTML is the key starting point toward good accessibility practices. When a screen reader, or any sort of assistive device scans a web page, it gets information about the Document Object Model (DOM), or the HTML structure of the page. No styles or JavaScript will be read by a screen reader. An excerpt from the Web Accessibility Guidelines v1.0 website.
